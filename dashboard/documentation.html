Documentation

This dashboard enables detailed evaluation and comparison of feature selection algorithms across multiple datasets, metrics, and feature selection ratios. It supports both built-in benchmark results and user-uploaded custom data.

Main Controls

  • Dataset: Select which dataset to visualize in the Performance Curves.
  • Metric: Choose from CLSACC (Classification Accuracy), NMI (Normalized Mutual Information), ACC (Clustering Accuracy), or AUC (Area Under Curve).
  • Range:
    • 10Percent: Fine-grained from 0.5% to 10% (step 0.5%)
    • 100Percent: Coarse-grained from 5% to 100% (step 5%)
  • Runtime Type (for Runtime plot): Toggle between Instances Experiment and Features Experiment.

Core Features

Performance Curves (Line Plot) Shows how each method's performance changes as more features are selected. Random method includes a shaded confidence band (±1 std). Horizontal gridlines aid readability.
Score Table Displays average performance per dataset. Best result is bolded, second-best is underlined. Supports direct copy-paste into documents.
LaTeX Table Generator Converts the score table into publication-ready LaTeX code, preserving bold and underline formatting.
Critical Difference Diagram Uses Nemenyi post-hoc test after Friedman test. Methods connected by a thick horizontal bar are not significantly different. Lower average rank = better.
Runtime Scalability Analysis Visualizes runtime vs. number of instances or features (log-scale y-axis). Helps assess computational efficiency and scalability of each method.

Data Format Requirements

Files must follow this exact naming convention:

Method_Metric_Range.csv
Example: VCSDFS_CLSACC_10Percent.csv

The CSV must contain:

  • A column named Dataset
  • Columns named with percentage values as strings (e.g., 0.005, 0.01, ..., 1.0)
  • Each row corresponds to one dataset with performance values at each percentage level
Note: Custom files are merged with built-in benchmark data. Duplicate method+metric+range combinations will be overridden by uploaded files.

Import Custom Data

Click the Import button (orange) to upload one or multiple CSV files. Supported formats are exactly as described above. After upload, the dashboard automatically refreshes to include your data alongside built-in benchmarks.

Exclude Datasets or Methods

Click Exclude (orange) to open the exclusion panel. Use Python-style list syntax:

DATASETS = ["Yale", "ALLAML"]
METHODS = ["Random", "Variance", "Laplacian"]

Click Apply Exclusion. The excluded items will be removed from all plots, tables, and diagrams until you clear the list or reload the page.

Runtime Scalability Analysis

This section shows how runtime grows with dataset size:

  • Features Experiment: Runtime vs. number of features (fixed instances)
  • Instances Experiment: Runtime vs. number of instances (fixed features)

Y-axis is logarithmic to better visualize wide runtime ranges. Missing values (-1) are automatically skipped. The plot is also downloadable as PDF.

Download Options

  • Download Line Plot (PDF) – Current dataset + metric curve with gridlines and legend
  • Download CD Diagram (PDF) – Critical difference diagram
  • Download Runtime Plot (PDF) – Scalability plot with single-line legend below

Tips & Best Practices

  • Use 10Percent range for fine-grained analysis in low-selection regimes.
  • Use 100Percent for overall performance overview.
  • Check the CD diagram to confirm statistical significance of ranking differences.
  • For publication, generate LaTeX tables and copy directly into your paper.
  • Runtime plots help identify methods that scale poorly with large datasets.
