Benchmarking with FSEVAL

The sdufseval library allows you to reproducibly benchmark feature selection methods on high-dimensional datasets and analyze their runtime scalability.

Installation

Install the package directly from PyPI:

Install via pip
pip install sdufseval

Full Benchmark + Runtime Analysis Example

This script runs:

  • Standard performance evaluation on 23 classic high-dimensional datasets
  • Runtime scalability experiments (varying number of features and number of instances)
benchmark.py
from sdufseval import FSEVAL import numpy as np if __name__ == "__main__": # The 23 benchmark datasets DATASETS_TO_RUN = [ 'ALLAML', 'CLL_SUB_111', 'COIL20', 'Carcinom', 'GLIOMA', 'GLI_85', 'Isolet', 'ORL', 'Prostate_GE', 'SMK_CAN_187', 'TOX_171', 'Yale', 'arcene', 'colon', 'gisette', 'leukemia', 'lung', 'lung_discrete', 'madelon', 'orlraws10P', 'pixraw10P', 'warpAR10P', 'warpPIE10P' ] # Initialize FSEVAL evaluator = FSEVAL( output_dir="benchmark_results", avg_steps=10 ) # Configuration for methods methods_list = [ { 'name': 'Random', 'type': 'unsupervised', 'stochastic': True, 'func': evaluator.random_baseline }, { 'name': 'Variance_Baseline', 'type': 'unsupervised', 'stochastic': False, 'func': lambda X: np.var(X, axis=0) } # Add more methods here (VCSDFS, LIDFS, SCFS, MCFS, etc.) ] # ─── 1. Run Standard Benchmark ─── # Evaluates methods on real-world datasets across different feature scales evaluator.run(DATASETS_TO_RUN, methods_list) # ─── 2. Run Runtime Analysis ─── # Performs scalability testing on synthetic data with a time cap. # vary_param='both' → runs both 'features' and 'instances' experiments print("\n>>> Starting Scalability Analysis...") evaluator.timer( methods=methods_list, vary_param='both', time_limit=3600 # 1 hour limit per configuration )

Output

After running the script, you will find:

  • benchmark_results/ folder containing CSV files with performance metrics (CLSACC, NMI, ACC, AUC) for each method–dataset–ratio combination
  • Runtime CSV files (time_analysis_features.csv and time_analysis_instances.csv) that can be directly visualized in the dashboard
